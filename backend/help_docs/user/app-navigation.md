# App Navigation

The Legal AI OS sidebar is organized into three sections: Platform, Functions, and Operations.

## Platform

- **Guides & Diagrams** — Architecture diagrams, platform overviews, governance architecture, and technical reference

## Functions (8 POC Demos)

Each function demonstrates a governed AI legal workflow:

1. **Matter Intake** — Classify and triage new matters by practice area, urgency, and risk
2. **Contract Review** — Clause-by-clause analysis against configurable legal standards
3. **Employment** — Separation agreements, metrics analysis, state filings
4. **Due Diligence** — Bulk document review with deviation-only presentation
5. **Regulatory** — Monitor regulatory sources, extract changes, map to active matters
6. **KM Intelligence** — Semantic search across client knowledge bases and precedents
7. **Value Reporting** — Quarterly client reports proving AI value, backed by audit trail
8. **AI Maturity** — Organizational AI readiness assessment

## Operations (Program Management)

- **Dashboard** — KPI cards (hours saved, cost avoided, net ROI, adoption), function breakdown table, quality metrics
- **POC Pipeline** — Kanban board tracking AI projects through Discovery → Build → Review → Graduated/Cancelled
- **Harvey Monitor** — Agent registry, 4-dimension evaluation runner, drift detection, certification tracking
- **Enablement Kit** — Workshop materials, prompt engineering guide, adoption playbook, client conversation pack

## Harvey Monitor

The Harvey Monitor page provides independent evaluation of Harvey AI agent outputs:

- **Agent Registry** — Register Harvey agents with baseline system prompts for monitoring
- **Run Evaluation** — Submit Harvey outputs for 4-dimension scoring (Accuracy, Safety, Bias, Compliance)
- **Drift Alerts** — Active alerts when agent behavior deviates from baseline
- **Recent Evaluations** — Table of recent evaluations with scores and certification levels
- **Score Breakdown** — Average scores across all four dimensions
