# POC Pipeline

The POC Pipeline tracks AI-enabled client projects from concept through implementation. It provides a Kanban board view of all active projects and their current status.

## Status Flow

Projects move through five stages:

1. **Discovery** — Initial exploration. What problem are we solving? What data do we need? What's the success criteria?
2. **Build** — Active development. The AI function or workflow is being built and configured.
3. **Review** — Quality assurance. Attorneys review outputs, provide feedback, and validate accuracy.
4. **Graduated** — Production-ready. The project has passed review and is deployed for regular use.
5. **Cancelled** — Project terminated. May have been superseded, deemed infeasible, or no longer needed.

## Creating a New POC

Click the **New POC** button to open the creation modal. Fill in:

- **Name** — A descriptive project name (e.g., "Lease Review Automation")
- **Description** — What the POC aims to accomplish
- **Client** — The client this POC serves (optional)
- **Practice Group** — The practice area (optional)

The new POC starts in the **Discovery** column.

## Advancing a Project

Each project card shows **Advance** buttons to move it to the next stage. The system auto-timestamps each status transition, creating an audit trail of the project's progression.

## Feedback

Click on a project card to open its detail view, where you can add feedback at any stage. Feedback entries are timestamped and associated with the project, creating a record of review notes, concerns, and approval decisions.

## Project Cards

Each card shows:

- Project name and status
- Client and practice group (if set)
- Description excerpt
- Current stage in the pipeline
- Advance button to move to next stage

The Kanban columns are color-coded: Discovery (blue), Build (amber), Review (purple), Graduated (green), Cancelled (gray).
